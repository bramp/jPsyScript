Notes: stimulus files

This page discusses some details about your options for organising your script file(s) and stimulus file(s) in folders.

PsyScript goes some way towards making it easy to keep track of the files you'll need for an experiment but it's possible to get yourself in a situation where you'll have to go through the script correcting every reference to a stimulus file. Reading this page should ensure that you never have to do this.

Simple experiments

A simple experiment would probably involve one script and some stimulus files. In this case it's simplest just to keep the script and the the associated stimulus files in the same folder. I'd recommend that you create another folder (possibly inside the first one) and keep all the log files in that.

If a script needs to refer to a stimulus file in the same folder as the script, all it needs to use is the file's name, like this:

play sound clapping.snd

More complicated experiments

A more complicated experiment might involve one script and a number of different classes of stimulus files. You might, for example, want to keep two kinds of stimuli in different folders. In this case it's simplest to keep the two folders in the same folder as the script. If you do this you can refer to files in the folder as follows:

play sound withEcho/clapping.snd

play sound withoutEcho/clapping.snd

You use a '/' sign to separate the name of the folder from the name of the file inside it. If you use this system then you can move your experiment files from one disk to another, or from one folder to another, and as long as you maintain the relationship between the places of the files (script in the folder which contains the folders where you keep your stimuli), your project will continue to work.

You can extend this further:

table faces

  female/age 6 to 10/jenna.pict

  female/age 3 to 5/cally.pict

  male/age 6 to 10/gan.pict

  male/age 3 to 5/avon.pict

  male/age 3 to 5/vila.pict

end table

with as many levels of folder as you need. You can use spaces in the names of folders and files.

Absolute locations

Under some circumstances you may need to refer to stimuli files which can't be 'below' the script in the folder hierarchy. In this case you'll need to use the full path to the stimulus files. You can spot paths like this because they begin with a '/':

play movie /Volumes/Chatting/kids/age 3 to 5/BlakeAndSoolin.mov at (0,0) with size (200,200)
