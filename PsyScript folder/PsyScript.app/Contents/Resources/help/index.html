PsyScript: an experiment generator.

This is the built-in help for PsyScript, an experiment generation package intended for use by psychologists and other researchers working in fields which involve running experiments on people. PsyScript is not a fully-featured programming language: a lot of effort was put into keeping it as simple as possible while providing features required for psychology experiments. PsyScript runs on any computer running OS X 10.3.9 or later.

If you intend to use PsyScript to run actual experiments, please read the page of legal claims and disclaimers.

PsyScript's homepage is http://www.psych.lancs.ac.uk/software/.

Files provided

  • The application (with built-in help pages)
  • A one-page PDF file with a list of all commands on it
  • Three tutorials as .PDF files
  • Some sample scripts (and the stimuli they need to run)
  • Some other sample stimuli

This application is written in Objective-C. It was developed using Apple's free development tools, Xcode and Interface Builder. HTML documentation was coded by hand. The tutorial was prepared in Keynote. Pictures, diagrams and example stimulus images were prepared with AppleWorks 6 (come on Apple, give us a drawing module for iWork !).

Issues with upgrading from version 1 to version 2

  • If, after upgrading from version 1.x, PsyScript is ignoring your preferences for the log file format, delete the preferences file. It's ~/Library/Preferences/com.univ.lancs.PsyScript.plist.

Known bugs in version 2.0.0

  • When you quit the program on a computer with the drivers for National Instruments cards loaded, it exits with error 1 instead of just calmly quitting. This is due to a bug in the National Instruments drivers and does not indicate any real problem, or that anything bad has happened while you were using PsyScript. NI tell me that the bug will be fixed soon.
  • You need to hit the tab key or click inside the text field before you can type in it. The program should be doing this automatically when the text field is shown. I'll fix this bug when I understand why it's happening.
  • The application and document icons are not very good. If you think you can do better, email me.

Syntax changes up to version 2.0.0

  • The load image command no longer works without the word 'image'.
  • The play movie command no longer accepts the syntax which includes a size for the movie area on the display. The movie is always displayed at whatever size it's saved with.
  • The wait for ? second[s] command has an alternative syntax pause for ? second[s].
  • A large number of new commands were added since version 1.5. Take a look at the 'Commands' window or at the online help to see them.
