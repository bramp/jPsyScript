Command groups: log

It's pretty pointless running an experiment if you don't take notes about what happened. That's what the log command is for. The format of the command is simple:

        log <value>

There are a number of different things you can log, but they split up neatly into three groups:

special use fixed text(anything that doesn't start with a '$') that text.
$returnStarts a new row of the log.
$dateStampThe current date in text form.
$timeStampThe current time in text form.
$spaceOne 'space' character.
$blankNothing. (Useful for the set command or to leave a blank column in the log.)
 
entered by
experimenter		$experimentName of, or code for, the experiment. Entered by experimenter just before log is saved into a file.
$subjectName of, or code for, the subject. Entered by experimenter just before log is saved into a file.
$commentsComments on this run of the experiment. Entered by experimenter just before log is saved into a file.
 
result of
commands		$lastClickThe name of the cell wait for a click accepted a click in last time it was used.
$lastClickTimeThe number of seconds wait for a click waited for last time it was used.
$lastKeyThe key wait for a key accepted last time it was used.
$lastKeyTimeThe number of seconds wait for a key waited for last time it was used.
$timerThe number of seconds since the reset timer command was used.
$countdownThe number of seconds until the countdown timer expires, or 'expired' if it has already expired.
$sliderThe current setting of the slider.
$textFieldThe current contents of the text field.
$var<letter>The value of that variable.

Although PsyScript doesn't enforce this, it's customary to lay out the log in a number of rows of equal length. This is especially useful if you expect to read the log file directly into a statistics program such as SPSS.

Sample

      proc main

        log $experiment

        log $subject

        log $return



        wait for a key

        log $lastKey

        log $lastKeyTime

        log $return

      end proc

Commands

log value
If <value> is '$return' then a new row in the log file is started. Otherwise the value specified is put in the next column of the current row of the log file.
Examples:log $slider
log The subject never clicked on any target.
log $return

Technical notes

You can change the row and column separators and the default extension for the script using the application's 'Preferences' menu item.
