Command groups: background

If you do not explicitly set the background colour in a script you will see the default OS X window background, a pattern of grey and white stripes. The background command allows you to set the background to of the experiment's window to any solid colour you like (but you can't use it to reset the background to the original stripe pattern, it has to be one solid colour).

There are two forms of the background cammand. The more flexible looks like this:

background (red-component,green-component,blue-component)

Each of the components is a number from 0 to 65535. 0 means that there is no element of that component in the mix, that that component is completely off; 65535 means that that component is full-on, as bright as it can be in the mix. Here are some examples:

parametersresulting colour
(0,0,0)black
(65535,65535,65535)bright white
(65535,0,0)bright red
(0,65535,0)bright green
(32768,32768,32768)mid-grey
(10000,20000,30000)a pleasant dark blue

You can, of course, use variables for any of the components:

background ($varR,$varG,$varB)

or even mix them:

background (23000,0,$varD)

For your convenience some commonly-used colours have been given text names. You cannot use variables containing these text values but they help to make simple scripts a bit easier to understand. These are the pre-defined names:

black, dark grey, mid grey, light grey, white

Commands

background (red-component,green-component,blue-component)
red-component -- red brightness from 0 to 65535
green-component -- green brightness from 0 to 65535
blue-component -- blue brightness from 0 to 65535
Sets the background colour of the experiment window.
Examples:background (32768,32768,32768)
background ($varA,0,0)

background predefined colour
predefined colour -- one of five pre-defined colours: white, light grey, mid grey, dark grey, or black
Sets the background colour of the experiment window.
Examples:background mid grey
background black

Technical notes

Depending on incident light, how your display works (e.g. what display card and monitor you're using), how you have ColorSynch set up, and how you have the 'brightness' control set, the parameters you supply are converted into a 'real' colour for display. Consequently, do not rely on a particular set of parameters representing a particular colour.

Don't rely on colour brightnesses being conserved across colours. For instance, (10000,0,0) will probably not be exactly as bright as (0,10000,0). If you need chromatic precision you must use a properly calibrated setup.

Don't rely on colour differences being conserved across the full range. For instance, (10000,0,0) may be a certain amount dimmer than (15000,0,0), but (20000,0,0) may not be exactly the same amount brighter again. If you need chromatic precision you must use a properly calibrated setup.
